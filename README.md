台灣 / 日本餐廳隨機推薦器
功能介紹

這是一個 網頁版餐廳隨機推薦工具，可以快速依據使用者選定的地區與餐廳類型，隨機抽選 1～3 家餐廳，並提供地圖查看、Google Maps 開啟、導航功能。

支援 台灣與日本地區。

可選 城市、行政區、街道。

可選 餐廳類型：餐廳、咖啡店、酒吧、麵包/烘焙、甜點、小吃、夜市、飲料等。

提供 半徑搜尋功能（手機版可取得定位後使用）。

顯示 店家營業時間與 地址來源（OSM、LocationIQ、Nominatim）。

支援 隨機重新抽選功能。

裝置差異
電腦版

使用方式：

選擇國家 / 縣市 / 區域 / 街道 / 店家類型。

設定搜尋半徑（0 表示全區搜尋）。

點擊「搜尋餐廳」隨機抽選。

不支援「取得我的位置」功能。

手機版

使用方式：

可手動選擇國家 / 縣市 / 區域 / 街道 / 店家類型。

或點擊 📍 取得我的位置，系統將以使用者當前位置為中心搜尋附近餐廳。

設定搜尋半徑（單位：公尺）。

點擊「搜尋餐廳」隨機抽選。

地圖與行動功能

每個餐廳結果提供三個按鈕：

📍 顯示在地圖：將該餐廳定位到地圖中心。

🗺️ 在 Google Maps 開啟：開啟 Google Maps 搜尋或座標。

🚗 導航：透過 Google Maps 導航到該餐廳。

iOS / Android 會自動使用原生地圖 App 開啟。

注意事項

資料來源包含 OSM、LocationIQ、Nominatim，部分店家可能缺少地址或營業時間。

若地址資料不足，會使用經緯度備援，Google Maps 會以座標開啟。

日本地區資料可能不完整，部分城市或餐廳資訊缺失，請以搜尋結果為準。

手機定位需授權瀏覽器存取位置。

搜尋半徑建議設定 500~3000 公尺，過大可能導致地圖或 Overpass API 回傳慢。

安裝與使用

下載或克隆專案

git clone <repository-url>


打開 index.html

建議使用 Chrome / Edge / Safari 瀏覽器。

不需後端伺服器即可運作。

設定 LocationIQ API Key

打開 script.js，將：

const API_KEY = "LocationIQ API Key";


替換成你自己的 LocationIQ Key。

地圖：Leaflet.js + OpenStreetMap tiles

地理編碼：LocationIQ + Nominatim

餐廳資料：Overpass API 查詢 OSM 資料

UI：

電腦版完整欄位顯示

手機版可折疊欄位，半徑搜尋與定位功能特化

主程式語言：HTML / CSS / JavaScript

支援 暗黑模式切換，狀態會記錄在 localStorage。

手機 UI 根據螢幕寬度自動調整。

支援重新搜尋、隨機抽取 3 筆餐廳、即時街道建議。

使用 防呆判斷，排除只有行政區或不完整地址的資料。
